# ClaudeMonitor - Документ требований к продукту

## Видение продукта
Компактное desktop-приложение (Windows) для мониторинга использования Claude AI и OpenAI Codex в реальном времени. Плавающее полупрозрачное окно поверх всех окон показывает процент использования, обратный отсчёт до сброса 5-часовых и недельных лимитов, статус инцидентов на стороне Claude, а также управление несколькими Claude-аккаунтами.

## Целевая аудитория
Активные пользователи Claude (claude.ai, Claude Code) и Codex, которым важно видеть текущий расход лимитов, понимать время до сброса квот и оперативно выбирать рабочий аккаунт/сессию.

## Основные функции

### 1. Мониторинг использования Claude (реализовано)
- Авторизация через CookieBridge (основной поток) или Playwright (fallback)
- Отображение процента использования по категориям: Сессия, Все модели, Claude Design/отдельный недельный лимит, Opus, Sonnet, Cowork, OAuth Apps
- Обратный отсчёт до сброса лимитов (обновление каждую секунду)
- Автообновление каждые 3 минуты
- Компактный режим (только процент сессии + время активного аккаунта)
- Иконки в трее с динамическим процентом выбранного аккаунта/сессии
- Сохранение позиции окна между перезапусками

### 2. Мониторинг использования Codex (реализовано)
- Единая таблица Claude/Codex без подписей разделов: слева от логина показывается брендовая иконка провайдера (Claude Code для Claude-аккаунтов, ChatGPT для Codex-строки), дальше общие колонки логина, тарифа, 5ч и 7д лимитов
- Отображение логина активной локальной Codex-сессии
- Отображение тарифа Codex/ChatGPT (например, Pro Lite)
- Отображение процента использованного 5-часового лимита (0-100)
- Отображение процента использованного недельного лимита (0-100)
- Обратный отсчёт до сброса 5-часового и недельного лимитов
- Источник цифр: официальный локальный `codex app-server` (`account/rateLimits/read`), то есть те же `usedPercent` и reset-время, которые использует сам Codex; чтение журнала `~/.codex/logs_2.sqlite` остаётся fallback; логин/тариф добираются из `account/read` и локального `~/.codex/auth.json`
- Автообновление Codex-строки раз в минуту и пересчёт обратного отсчёта каждую секунду
- Правая галочка в любой строке общей таблицы выбирает, какие 5ч/7д проценты показывать на двух иконках трея; выбор сохраняется в `accounts_meta.json`

### 3. Мониторинг инцидентов Claude Status (реализовано)
- Опрос Atlassian Statuspage API (`status.claude.com/api/v2/incidents/unresolved.json`) каждые 2 минуты
- Кликабельные лейблы активных инцидентов внизу окна (цвет по severity)
- Попап с деталями инцидента по клику на лейбл
- Кастомные toast-уведомления при появлении/завершении инцидентов

### 4. Мультиаккаунтность Claude (реализовано)
- Поддержка до 4 аккаунтов claude.ai
- Хранение куков каждого аккаунта в `accounts/<id>.json`
- Таблица аккаунтов в развёрнутом окне: иконка провайдера, email/имя, тариф, % сессии (5ч), % недельного лимита (7д), % отдельного недельного лимита Design/модели, левая кнопка активного Claude-аккаунта и правая кнопка источника трея
- Активный аккаунт отмечен зелёной точкой
- Горячее переключение — без перезапуска
- Фоновый поллинг всех аккаунтов каждые 5 минут
- Автоопределение email/имени аккаунта через API после логина
- Кнопка "+ Добавить" в секции аккаунтов
- Переключение через контекстное меню (правая кнопка)

### 5. CookieBridge интеграция (реализовано)
- Локальный HTTP-сервер на порту 19225 (запускается вместе с приложением)
- Эндпоинт `POST /cookies` — получает HMAC-подписанные куки от CookieBridge Hub
- Эндпоинт `GET /health` — статус для popup расширения
- При получении куков: автоматически определяет аккаунт по `sessionKey`, сохраняет, устанавливает активным если нет других
- CookieBridge Hub хранит также куки `chatgpt.com` для целевой Codex-сессии; текущая реализация Codex-лимитов использует локальный журнал Codex как более прямой источник чисел лимитов

### 6. Интеграция с TrayConsole (реализовано)
- Управление через Named Pipes: show/hide/refresh/relogin/shutdown
- Heartbeat для мониторинга процесса

### 7. Поддержка HTTP/HTTPS прокси (реализовано)
- Файл `proxy.json` рядом с приложением: `{"url": "http://127.0.0.1:10808"}`
- Fallback: переменные окружения `HTTPS_PROXY` / `HTTP_PROXY`
- Прокси применяется ко всем исходящим запросам: curl_cffi subprocess (fetch/identity/ping/login) и urllib (Statuspage)
- Пункт контекстного меню "Прокси: &lt;url&gt;" открывает диалог для смены URL (пустое значение — выключить прокси)
- Нужен, когда claude.ai/status.claude.com недоступны напрямую и доступ идёт через локальный прокси

## Технический стек
- Rust + Slint (активная desktop-реализация)
- reqwest/rustls (HTTP для API claude.ai, Statuspage и CookieBridge Hub)
- axum (локальный CookieBridge receiver на 19225)
- tray-icon + Windows Named Pipes (трей и TrayConsole)
- Tokio (фоновые обновления Claude/Codex/Statuspage)
- Legacy Python/PyQt5 файл `usage_monitor.py` сохранён в репозитории, но текущий запуск через `start.bat` использует Rust release-бинарь

## Технические ограничения
- Только Windows для production-запуска (TrayConsole/Named Pipes/tray)
- Rust release-бинарь запускается через TrayConsole (`trayconsole.json` + `start.bat`)
- Statuspage API публичный, без аутентификации
- Chrome хранит куки в едином хранилище профиля — два аккаунта одновременно возможны только в разных профилях Chrome

## Структура хранения аккаунтов
```
accounts/
  acc_<md5>.json     # Playwright storage_state с куками
accounts_meta.json   # { active: "acc_...", accounts: [{id, email, name, added_at}] }
```
